Bucle While

Estructuras de control.

El bucle while comienza evaluando la condición. Si la condición es verdadera, se ejecuta el bloque de código dentro del bucle. Después de ejecutar el bloque de código, la condición se vuelve a evaluar. Si la condición sigue siendo verdadera, se repite el ciclo y se vuelve a ejecutar el bloque de código. Este proceso continúa hasta que la condición se evalúa como falsa, momento en el que se sale del bucle y la ejecución del programa continúa con la siguiente línea de código después del bucle.

4

Bucle Do While

Estructuras de control

El bucle do-while primero ejecuta el bloque de código y luego evalúa la condición. Esto significa que el bloque de código se ejecutará al menos una vez, sin importar si la condición inicialmente. Después de ejecuta el bloque de código, y evalúa la condición. Si la condición es verdadera, el bucle se repite . El ciclo continúa hasta que la condición se evalúa como falsa, se sale del bucle y la ejecución del programa continúa con la siguiente línea de código después del bucle.

5

Bucle FOR

Estructuras de control

1- Inicialización:
Inicializa una variable de control del bucle. Se ejecuta solo una vez al inicio del bucle.
2- Condición:
Se evalúa antes de cada iteración del bucle. Si se evalúa como verdadera, el bloque de código se ejecuta. Si se evalúa como falsa, sale del bucle.
3- Actualización:
Se utiliza para actualizar la variable después de cada iteración. incrementa o decrementa su valor.

6

Bifurcaciónes / Condicionales
( IF -ELSE - ELSE IF )

Estructuras de control

La condición del if debe ser una expresión que se evalúa como un valor booleano: Si la condición es verdadera, ejecuta el bloque de código que sigue al if. Si la condición es falsa, se omite ese bloque y se continúa con la ejecución del código. Opcionalmente, se puede utilizar la estructura else para especificar un bloque de código alternativo que se ejecutará si la condición es falsa
Además, se puede encadenar múltiples condiciones utilizando la estructura ELSE IF

7

Estructura SWITCH

Estructuras de control

La estructura switch en JavaScript es una forma de control de flujo que permite tomar decisiones basadas en el valor de una expresión. Se utiliza cuando se necesita evaluar una expresión y ejecutar diferentes bloques de código dependiendo del valor que tome esa expresión.

Tipos dinámicos JavaScript

Interpretación de lectura de datos

En JavaScript, los tipos de datos son dinámicos, lo que significa que una variable puede contener diferentes tipos de datos a lo largo de su vida útil. A diferencia de otros lenguajes de programación, no es necesario declarar explícitamente el tipo de una variable en JavaScript.

Propiedad y método

Interpretación de lectura de datos

Propiedades:
Las propiedades son valores asociados a un objeto. Representan características o atributos del objeto y se acceden utilizando la notación de punto o la notación de corchetes.

Métodos:
 Los métodos son funciones asociadas a un objeto. Representan acciones que el objeto puede realizar y se invocan utilizando la notación de punto.

Recursividad en JavaScript

Interpretación de lectura de datos

La recursividad en JavaScript es una técnica en la que una función se llama a sí misma durante su propia ejecución.
La recursividad permite resolver problemas complejos dividiéndolos en subproblemas más pequeños y más simples. En cada llamada recursiva, la función se resuelve para un caso base más simple y luego combina los resultados para obtener la solución del problema original.
